# LISTR 快速上手(写给不写代码的你)

[English version](quickstart-en.md) · [进阶指南 →](advanced-zh.md)

你不需要会编程。整个流程只有三步:**装好环境(一次性)→ 填一张
Excel 配置表 → 运行一行命令**。之后每次换数据,只改配置表就行。

## 第 0 步:装环境(只做一次,约 10 分钟)

1. 安装 R:打开 <https://cran.r-project.org>,选择你的系统
   (Windows / macOS),下载并一路"下一步"安装。
2. 安装 RStudio(更友好的界面):<https://posit.co/download/rstudio-desktop/>,
   同样一路"下一步"。
3. 打开 RStudio,在左下角 **Console(控制台)** 里粘贴下面两行,
   按回车,等它跑完:

   ```r
   install.packages("remotes")
   remotes::install_github("weiandata/LISTR")
   ```

   > 如果公司内网有现成的安装包,按 IT 同事给的方式装即可。

## 第 1 步:生成并填写配置表

在控制台粘贴、回车:

```r
LISTR::lst_config_template("我的配置.xlsx")
```

你的"文档"或当前工作目录下会出现 **我的配置.xlsx**。用 Excel
打开,先读"说明"工作表,然后照着灰色示例填另外四张表:

| 工作表 | 填什么 | 示例 |
| --- | --- | --- |
| 数据与角色 | 数据文件在哪、哪列是编号/权重/分组 | 数据文件 `students.csv`,分组列 `region,gender` |
| 能力维度 | 能力值列和标准误列(没有就留空) | math / th_math / se_math |
| 统计表 | 你想要的表:行、列、算什么 | 行 `region`,列 `gender`,统计量"平均能力" |
| 输出 | 结果存成什么文件 | `结果.xlsx` |

填写小贴士:

- **列名要和数据文件里的完全一致**(区分大小写)。
- 一行 = 一个统计量;同一"表名"的几行会合成一张透视表。
- "等级分数线"这样写:`待提高=-Inf,合格=40,良好=60,优秀=80`
  (等级名=下界,逗号分隔;`-Inf` 表示负无穷,即最低档没有下限)。
- "方法"填 `prob` 表示利用每个人的标准误做概率化计算(推荐,
  需要有标准误列);留空则按分数硬划分。

## 第 2 步:运行

回到 RStudio 控制台,粘贴、回车:

```r
LISTR::lst_run("我的配置.xlsx")
```

几秒钟后,配置里写的输出文件就生成了。

## 第 3 步:读结果

打开生成的 **结果.xlsx**:

- 每张统计表一个工作表,单元格形如 **`52.62 (0.02)`**——前面是
  估计值,**括号里是标准误**,表示这个数字的不确定程度。
  粗略判断:两个组的差距超过两边标准误之和的 2 倍,才值得当成
  真差异说。
- 最后有一张 **"结论"** 工作表,是自动生成的文字解读(最高最低
  组、是否显著、样本过小警告),可以直接引用到报告里。

## 出错了怎么办

LISTR 的报错都是中文,直接告诉你哪里要改,例如:

> 第 1 张表 的统计量 "优秀率" 缺少数值型 cutoff(阈值)。

按提示补上配置表里对应的格子,重新运行第 2 步即可。常见问题:

| 报错关键词 | 原因 | 办法 |
| --- | --- | --- |
| 找不到数据文件 | 路径写错或文件不在当前目录 | 把数据文件和配置表放同一文件夹,或写完整路径 |
| 中的列在数据里不存在 | 列名拼写和数据不一致 | 打开数据文件核对表头,复制粘贴列名 |
| 缺少工作表 | 配置表的 sheet 被改名/删除 | 重新生成模板,把内容搬过去 |
| 暂不支持的文件格式 | 数据扩展名不对(如 `.sav_副本`) | 把文件重命名为规范的 `.sav`/`.csv` 等 |
